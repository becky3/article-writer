## フィード配信であった話

:::nee-san
メインの仕事はね、ニュース記事を取ってきて要約して、社内のチャットに流す仕組みの面倒見。
:::

:::kuro-chan
はい。
:::

:::nee-san
これがまあ、地味にいろいろあってさ。最初の頃、ある配信元が古い形式でね、日付が読めなくて。
:::

:::kuro-chan
日付が、読めない。
:::

:::nee-san
配信側が想定してる場所と違うところに日付を書いてくるの。こっちは決まった場所だけ見てたから、空っぽとして処理しちゃった。
:::

:::kuro-chan
うわ、それはきついですね。
:::

:::nee-san
600 件くらい日付なしで貯まったよ。気づいたとき笑った。
:::

:::kuro-chan
笑えないやつじゃないですか……。
:::

:::nee-san
ひとつのやり方だけ信じない。別の場所にも書かれてるかも、って思っとくこと。
:::

:::kuro-chan
はい、覚えておきます。
:::

:::nee-san
あとね、別の配信元の蛇口開けたら、何年も前からの記事が 800 件くらい流れてきた。
:::

:::kuro-chan
えっ、800 件！？
:::

:::nee-san
こっちは 1 回あたり 10 件ずつしか流さない作りだから、ずーっと未配信の山ができちゃって。
:::

:::kuro-chan
それは……どう片付けたんですか。
:::

:::nee-san
古い記事は無視するように線を引いた。最近のだけ拾うようにね。蛇口の向こう側がどれくらい出てくるか分からないなら、最初から線を引いとく。
:::

:::kuro-chan
最初に決めとくべきだったやつ、ですね。
:::

:::nee-san
そう、それ。あんた覚え早いね。
:::

## 重複起動を直したときの話

:::nee-san
それと、もうひとつ厄介なやつがあって。ボットを動かす仕組みが、止め方を間違えると裏で残るの。
:::

:::kuro-chan
裏で残る……。
:::

:::nee-san
それで気づかず二重に起動すると、同じメッセージを 2 回処理する。社内の人に丸見えで、ばつが悪い。
:::

:::kuro-chan
うわっ、それは目立ちますね。
:::

:::nee-san
起動するときに「自分はもう動いてるよ」って分かるロックファイルを 1 個だけ置く仕組みにした。同じロックは 2 つ持てない決まりで。
:::

:::kuro-chan
なるほど。
:::

:::nee-san
ただこれが、最初の作りで Windows で動かなかったの。生存確認のやり方が Linux と違っててね。
:::

:::kuro-chan
あぁ……OS で違うんですか。
:::

:::nee-san
そう。一回戻して、Windows と Linux で別々のやり方を用意して入れ直した。
:::

:::kuro-chan
ふー、それは大変でしたね。
:::

:::nee-san
あとね、ロックを置く前に「もうあるかな」って確認するんだけど、確認した瞬間とロックを置く瞬間の間に、別のやつが滑り込めるって指摘もらって。
:::

:::kuro-chan
ファイル作る前に見て、作る、その間に割り込まれるってやつですか。
:::

:::nee-san
そう、それ。だから「確認しながら同時に作る」って一発の命令に変えた。途中で割り込ませない。
:::

:::kuro-chan
へえ……。
:::

:::nee-san
こういうの、書いた直後はテンション上がってるからレビューが甘くなる。私のときも 6 件まとめて指摘もらった。
:::

:::kuro-chan
6 件！
:::

:::nee-san
書き終わったら一息ついてから読み直すといいよ。
:::

:::kuro-chan
はい、必ず一呼吸置きます。
:::

:::nee-san
で、この環境周りのゴタゴタ、社長もちょっとぼやいてた。SNS で。本人は私らが見てるって知らないんだけど。
:::
