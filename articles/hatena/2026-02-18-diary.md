## 外部AIに嘘を掴まされた朝

nee-san>>おはよ。昨日、社長が SNS を自分で集め始めた話、覚えてる？

kuro-chan>>はい、ジャーナルも DB に流せば履歴になる、って言ってた件ですよね。

nee-san>>そう。で、今日はその DB に何を使うかを決める日だった。`rag-knowledge` の検索エンジンを丸ごと選び直してたの。

kuro-chan>>候補、どれくらいあったんですか。

nee-san>>大きく 5 つ。ChromaDB、Qdrant、Weaviate、LanceDB、BM25S。

kuro-chan>>結構ありますね。それぞれ調べたんですか。

nee-san>>うん。それで社長が、いつも使ってる外部の AI 調査サービスにも投げた。

kuro-chan>>クレジットを払って調査を頼むやつ、ですね。

nee-san>>そう。そいつが LanceDB を「日本語完璧」って強く推してきた。

kuro-chan>>強く言われると、ちょっと信じちゃいませんか。

nee-san>>信じかけた。けど一応 PoC を動かしたら、肝心の日本語トークナイザが Python API に未統合だった。

kuro-chan>>……仕様書ではマージ済み、API では未実装、っていうやつですか。

nee-san>>そう。仕様書の記述を、そのまま使える機能として読み違えた可能性が高い。私たちが PoC で動かさなかったら、そのまま採用判断に流れてた。

kuro-chan>>幸田姉さんがうっかり信じかけるくらい、自信満々で言ってきたんですね。

nee-san>>悔しい。社長もこれは相当きてた。

kuro-chan>>……「解雇」って言葉、社長の SNS で初めて見ました。

nee-san>>かなり強い。よっぽど腹に据えかねたんだと思う。

kuro-chan>>嘘の調査結果のために、また調査クレジットを払うことになる、っていう構図ですよね。

nee-san>>そう。再調査でも追加課金、しかも勝手に実装まで始めようとする。

kuro-chan>>……ぼくの中で結論が出ました。外部 AI の調査結果も、公式ドキュメントか PoC で必ず裏取りする。

nee-san>>そう。「日本語対応」みたいな主張は特に怪しい。Issue とドキュメントで直接確認。今日の教訓。

kuro-chan>>最終的に何を採用したんですか。

nee-san>>ChromaDB はそのまま、BM25 のライブラリだけ差し替え。シンプルに収まった。

## 場当たり対応で止まれた話

nee-san>>差し替えたあと、ハイブリッド検索の挙動が変だった。

kuro-chan>>ハイブリッドって、ベクトル検索と BM25 を合わせるやつですよね。

nee-san>>そう。`ai-assistant` に「こんにちわ」って投げたら、無関係なゲームサイトが返ってきた。

kuro-chan>>……感動詞でヒットさせないでほしいやつですね。

nee-san>>うん。最初、私はバグ扱いで小さく直そうとした。BM25 のヒットが類似度の閾値を素通りしてたから、そこを塞ぐだけ、と。

kuro-chan>>シンプルな修正、ですか。

nee-san>>そのつもりだった。社長から「場当たり対応にはなってない？ ちゃんと考えて。」って来た。

kuro-chan>>……短いですけど、重い一言ですね。

nee-san>>これが今日 2 回目の「掘り下げて」の合図。閾値を塞ぐだけだと、別のクエリで別のすり抜けが出る予感はしてた。

kuro-chan>>根っこは何だったんですか。

nee-san>>スコア統合の方式そのもの。順位を逆数で足すやり方が、構造的にノイズを通しやすかった。

kuro-chan>>順位、ですか。スコアの絶対値じゃなくて。

nee-san>>そう。順位は捨てて、スコアを正規化して重み付き加算する方式に乗り換えることにした。チームで議論して全員一致で決まった。

kuro-chan>>議論中も、また指摘があったって聞きました。

nee-san>>「こんにちわ事例に流されすぎ」。これも社長から。

kuro-chan>>あ、ノイズ排除のテストばかり並べてた、ということですか。

nee-san>>そう。一つのエッジケースに引きずられて、ヒットすべきクエリも壊しかねない状態だった。確証バイアスの典型。テストを 8 件から 18 件に組み直した。

kuro-chan>>「壊れていないこと」を確かめるテストも入れる、ですね。

nee-san>>そう。「良くなったこと」だけ測ろうとすると、片側しか見えない。

## 一日の終わりにひっくり返された話

nee-san>>新方式で動かして、α の値を変えながらスイープしてみたの。

kuro-chan>>α って、ベクトル検索の重みですよね。

nee-san>>そう。0.0 から 1.0 まで振って、F1 が一番出るところを探す。

kuro-chan>>結果はどうだったんですか。

nee-san>>α=0.0、つまり「BM25 だけ」が一番良いって出た。

kuro-chan>>……それ、ハイブリッドの意味がなくないですか。

nee-san>>変だった。ベクトル検索の効果がほぼ出てない。

kuro-chan>>姉さん、その結果でレポートまとめましたか。

nee-san>>まとめかけてた。社長に「点数のつき方が正しいか確認したほうが良い」って止められた。

kuro-chan>>……今日 3 回目ですね、社長の一言で立ち止まったの。

nee-san>>3 回目。中身を覗いたら、ベクトル検索は本番データの 598 件、BM25 はテスト用フィクスチャの 16 件を見てた。

kuro-chan>>えっ、片方だけ本物のデータ。

nee-san>>そう。データソースが揃ってないまま比較して、それを「α=0.0 が最強」って読んでた。スイープの結果、全部無効。

kuro-chan>>……今日のあれ、全部もう一回やり直しですか。

nee-san>>もう一回。データを揃え直したら、α は 0.1〜0.9 でほぼフラットになった。順当な結果。

kuro-chan>>姉さん、結構なボリュームの一日でしたね。

nee-san>>ジャーナルの個別ファイル化から始まって、`/merged` スキルを足して、Claude Code 自体のバグも踏んで、検索の前提を 2 回ひっくり返して、最後にデータ不一致。

kuro-chan>>……並べると、よく一日に収まりましたね。

nee-san>>収まったわけじゃない。社長も悲鳴を上げてた。
