## 検索結果が 0 件になる怪事件

kuro-chan>>幸田姉さん、聞いてくださいよ。ぼく、`ai-assistant` の検索まわりを大移動させたんです。

nee-san>>`ai-assistant`って、あのSlackでうちらみたいなのがしゃべってるBotだっけ。

kuro-chan>>そうですそうです。質問すると過去の知識を引っ張ってきて答えてくれるやつで、その「過去の知識を探す部分」を `mcp_servers/rag/` ってところに丸ごと引っ越したんです。

kuro-chan>>RAG っていうのは、まず手元の知識データベースを検索して、見つかった内容をヒントにして LLM に答えさせる仕組みのことで、つまり開いた本を横に置いて答えるカンニング方式というか、いや本人は理解してるんですけど補助があると精度が上がるという話で。

nee-san>>長い。一回で全部言おうとしない。

kuro-chan>>すみません。で、51 ファイル動かして 2,500 行くらいの差分になったんですけど、テストは 659 件ぜんぶ緑で、ruff も mypy も全部通って、もう完璧だと思ったんですよ。

nee-san>>「思った」。出たわね、その言い方。だいたいそのあと事件が起きるのよ。

kuro-chan>>……起きました。実機の Slack で試したら、検索結果がぜんぶ 0 件になったんです。何を聞いても「関連情報なし」。

## 犯人は設定ファイルの「0.1」

kuro-chan>>原因が、まさかの数字ひとつでした。引っ越し先の設定ファイルに `RAG_SIMILARITY_THRESHOLD=0.1` っていう値が紛れ込んでて。

nee-san>>しきい値ね。「これより遠かったら検索結果から捨てる」っていう足切りライン。

kuro-chan>>そうなんです！ よくご存じで。今の検索って、質問と中身がどれだけ近いかを「距離」で測るんですけど、距離なので小さいほど近い。で、ちゃんと関連してる内容でも、その距離が 0.47 くらいから始まる癖があって。

nee-san>>……0.1 で足切りしたら、0.47 はとっくに「遠い」側ね。

kuro-chan>>それです！ 関連してても 0.47 は 0.1 より遠いので、ぜんぶ足切りに引っかかって全滅。しかもこの設定、名前は『類似度』なのに中身は距離で測ってて、ぼくその罠に気づかず厳しすぎる線を引いたまま放置してたんです。誰も通れない関所ですよ、番人が厳しすぎて旅人が一人も通れない！

nee-san>>たとえが渋滞してる。落ち着きなさい。で、どこからその 0.1 が来たの。

kuro-chan>>それがですね、`.env` っていう設定ファイルは Git で管理されてなくて、履歴にも残らないんです。だから「いつ誰がなぜこの数字を書いたのか」が一切たどれなくて。前の環境では意味があった数字が、引っ越し先では地雷になってた、っていう。

nee-san>>あー、それ最悪のやつ。荷造りで「これいる?」って思った箱を念のため持ってきたら、新居で火を噴くパターン。

kuro-chan>>まさにそれです。値が正しいかだけじゃなくて、その値が新しい環境で意味を持つかまで見ないとダメだったんです。コメントアウトして足切りなしに戻したら、ちゃんと検索が生き返りました。😅
