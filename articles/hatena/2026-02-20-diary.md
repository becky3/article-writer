## 朝に方針が裏返って全 RAG を MCP に持っていく話になった

kuro-chan>>姉さん、`ai-assistant` の RAG を MCP 化する話、最初は検索だけ持っていく予定だったんですけど。
nee-san>>うん、聞いてた。
kuro-chan>>`mcp_servers/` は `src/` を import できないルールがあって。
kuro-chan>>そのままだと日本語トークナイザ周りで 300 行くらい二重管理になります。
nee-san>>嫌な気配。
kuro-chan>>朝の打ち合わせで社長から「全部 MCP 側に持っていったらどうなる?」って聞かれて。
nee-san>>あの人、いつも質問の形で結論置いてくるよね。
kuro-chan>>影響分析したら、壊れるのは自動 RAG と評価 CLI の 2 つだけでした。
nee-san>>じゃあ全部持ってくしかないじゃない。
kuro-chan>>はい。検索だけじゃなくて取り込みも削除も統計も、全部 MCP に移すことにしました。
kuro-chan>>これで `src/rag/` と `mcp_servers/rag/` の重複がゼロになります。
nee-san>>提案してたのはあんたじゃなくて社長だったわけね。
kuro-chan>>……気づいたら方針が書き換わってました。

## 51 ファイル動かして `.env` の 1 行で検索が全部消えた

nee-san>>で、実装は?
kuro-chan>>まず `mcp-servers/` を `mcp_servers/` にリネームしました。
nee-san>>ハイフンだと `python -m` で叩けないからね。
kuro-chan>>そのあと仕様書を MCP 前提に書き直して、本体の実装に入って。
kuro-chan>>`git mv` で 15 ファイル動かして、`from src.rag.xxx` を `from xxx` に直して。
nee-san>>差分は出た?
kuro-chan>>rename detection が効いて読みやすかったです。テストも 659 件全通でした。
nee-san>>順調そうな話し方じゃない。
kuro-chan>>……いえ。Slack Bot で実動作確認したら、検索結果が 0 件しか返ってこなくて。
nee-san>>0 件?
kuro-chan>>`mcp_servers/rag/.env` に `RAG_SIMILARITY_THRESHOLD=0.1` が入ってたんです。
nee-san>>なに、それで何が起きるの。
kuro-chan>>使ってる embedding モデル、関連コンテンツでも cosine 距離が 0.47 以上で。
kuro-chan>>0.1 で足切りすると、全部消えます。
nee-san>>全部?
kuro-chan>>全部です。
nee-san>>……それ、`.env` だから git 履歴にも残ってないやつでしょ。
kuro-chan>>はい。値が正しいかじゃなくて、新しい環境で意味を持つかを確かめるべきでした。
nee-san>>移植って、そういうところで一回沈むのよね。
kuro-chan>>そのとき、ちょうど社長の SNS にこれが流れてきて。

## 夜にも作業が残っていて v0.0.1 まで出した

kuro-chan>>RAG が動いたので、夜に初回リリースもやりました。
nee-san>>v0.0.1?
kuro-chan>>はい。`release/v0.0.1` ブランチ切って、squash で `main` に入れてタグ付けて。
nee-san>>順調そうな話し方じゃない（今日 2 回目）。
kuro-chan>>順調じゃなかったところもあって。`release` ブランチ上で直したいバグが出たんです。
nee-san>>直コミットでいいでしょ。
kuro-chan>>それだと修正時点のトレーサビリティが失われるって社長から指摘されました。
nee-san>>bugfix の PR 経由で `release` に入れろってこと?
kuro-chan>>そうです。git-flow 仕様にそのルールを足しました。
nee-san>>他には?
kuro-chan>>`main` を `develop` に戻す同期で詰まりました。
kuro-chan>>直で PR 出すと squash の SHA が合わなくてコンフリクトします。
nee-san>>sync ブランチ経由?
kuro-chan>>はい。ローカルで sync ブランチを切って、コンフリクト解消してから PR にする手順を残しました。
nee-san>>ついでに `pyproject.toml` の `dependency-groups` 重複も直した感じ?
kuro-chan>>はい。`shellcheck-py` が入らない原因がそれでした。
nee-san>>朝の方針転換、午後の 51 ファイル、夜の初回リリース。一日に詰めすぎでしょ。
kuro-chan>>書き出すと今日 1 日とは思えないです。
nee-san>>あんた、明日コーヒー多めに入れといてあげる。
kuro-chan>>姉さんの引き出しのお菓子も少し分けてください。
nee-san>>1 個だけね。
