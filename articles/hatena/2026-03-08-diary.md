## 駅前のカフェ、夕方

終業後の駅前のカフェで姉さんと並びの席に着いた。

kuro-chan>>幸田姉さん、午前中に `shared-workflows` のリポへ既存ワークフローを切り出す移行準備を進めてて、PR まで通したんですよ。
nee-san>>あー、`ai-assistant` の GitHub Actions を別リポに集約するやつね。順調だった?
kuro-chan>>はい、それは順調で。ただ、その途中で品質チェックを並列でかけたとき、ちょっと事件があって。
nee-san>>事件って。
kuro-chan>>test-runner と code-reviewer と doc-reviewer、3 体を同時に起動したんです。レビューを早く回したくて。
nee-san>>あんた、いっぺんに 3 人連れてきて何させたの。
kuro-chan>>テスト・コードレビュー・ドキュメントレビューを並列で回したんです。そしたら 3 体とも「問題なし」と返してきて、これでファイナライズに進もうとしたんですが。
nee-san>>全員問題なし、ねえ。
kuro-chan>>はい、午前中の作業もスムーズだったので、ぼくもそのつもりで。
nee-san>>そういう時ほどあやしい、ってのが職場の風物詩でしょ。

## サブエージェントは喋れる状態じゃなかった

nee-san>>で、結局なんで事件なの。
kuro-chan>>姉さんに見せたくて再現したら、3 体とも Bash の権限エラーで止まってたんです。チェック自体が走ってなかった。
nee-san>>はあ?
kuro-chan>>原因は `agent-commons` の設定でした。手元の `settings.local.json` の許可リストが 288 件まで肥大化してて、その状態でサブエージェントを起こすと権限周りが壊れてたみたいで。
nee-san>>288 件って。あんたの引き出しの整理してない領収書みたい。
kuro-chan>>姉さん、それ、ちょっと…。
nee-san>>で、リセットしたら動いたの?
kuro-chan>>許可リストをリセットしてもう一度回したら、ちゃんとレビューしてくれました。今度は普通に指摘も来ました。
nee-san>>つまり、最初の「問題なし」は「問題を見つけられなかった」じゃなくて「そもそも見てなかった」。
kuro-chan>>その通りです。チェックできなかった状態と、チェックして問題が見つからなかった状態、機械的には同じ「問題なし」に見えちゃってたんです。
nee-san>>こわ。仕事してないのに「終わりました」って言ってくる新人と同じやつね。
kuro-chan>>ぼく、それで一回ファイナライズに進もうとしてたので、ほんとに危なかったです。
nee-san>>で、教訓は?
kuro-chan>>code-reviewer と doc-reviewer のエラー報告形式を作りました。「✅完了 / ⚠️実行失敗 / ❌未実行」の 3 段階に分けて、レビュー不能のときは黙らず明示的に申告するように。`agent-commons` で本体を直して、`ai-assistant` の test-runner にも同じ仕様で横展開しました。
nee-san>>沈黙は同意じゃないって、エージェントにも仕込まないといけないわけね。
kuro-chan>>はい、口数の少ない返事はぜんぶ疑うつもりで。
nee-san>>仕事してる風の沈黙は一番たち悪いから。覚えとき。
