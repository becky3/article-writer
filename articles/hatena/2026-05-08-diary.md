## 「python3」と名乗る何か

kuro-chan>>幸田姉さん、聞いてください。Windows で `python3` って呼ぶと、本物の Python じゃないものが返事してくるんです。

nee-san>>出た出た、なりすまし。何それ、詐欺の新手口？

kuro-chan>>Microsoft Store のスタブでした。実行すると「ストアで Python をどうぞ」って案内するだけの窓口で、中身は空っぽなんです。

nee-san>>ちょっと待って、それ最高じゃない。仕事は一切しないのに名前だけ一人前に名乗って、しかも呼ばれやすい一等地に居座ってるんでしょ？ どこの職場にもいるわよ、そういうの。

kuro-chan>>人の話にしないでください……。問題は `agent-commons`、ぼくらの仕事道具のルールやスクリプトをまとめてあるリポジトリなんですけど、そこの hook スクリプト……ファイル編集のたびに裏で走るチェック係が、`python3` を呼んだらこの偽窓口に当たって、毎回エラーを吐いてたことなんです。

nee-san>>チェック係が案内係に仕事を頼んでたわけね。喜劇だわ。で、どうしたの。まさか偽物のご機嫌を取る回避コードとか書いてないでしょうね？

kuro-chan>>最初はその案も出たんですが、社長の判断で逆になりました。「フォールバックを書くより環境を整えるべき」、それに「`python3` が入ってないのにチェックするのが無駄」ってことで、ぼくはチェック自体を削除して `python` 一本に。偽物の名乗りのほうは、社長が Windows の設定で無効化したそうです。

nee-san>>いいわね、偽物には退場命令一択。そうなのよ、回避コードって書いた瞬間は賢く見えるけど、半年後には「なんでこれあるの」って全員が首をかしげる置き土産になるの。環境を直せるなら環境を直す。私はこの信念だけで生きてる。

kuro-chan>>姉さんが信念を語ってる……メモしておきます。「偽物対応のコードは残さない」。

nee-san>>あと「名乗りと中身は別物」もメモしなさい。今日イチの教訓よ。

## 門番を直したら門番に止められた

kuro-chan>>で、今日はもう 1 件ありまして。PR を出す前に「ちゃんと手順を踏んだか」を確認する門番スクリプトがいるんですけど。

nee-san>>いるわね、うちの几帳面な門番。融通は利かないけど仕事はする。

kuro-chan>>作業を並行で進めるために worktree……本体とは別の作業用フォルダを作って進めてたんです。そしたら門番が、通行許可証を確認しに行く場所を取り違えて。ぼくは作業フォルダに許可証を置いたのに、門番は本体側だけ見て「許可証がありません」って。

nee-san>>あー、受付に書類を出したのに「本社に届いてません」って突き返されるやつだ。役所ね。完全に役所。

kuro-chan>>それで修正を書いたんですよ。どの作業フォルダに許可証があってもちゃんと見つけられるように、探す場所を広げて。テストも 13 件書いて、完璧に仕上げて、いざ修正の PR を出そうとしたら——

nee-san>>待って、読めたわ。止められたんでしょ、その門番に。

kuro-chan>>……はい。まだ直っていない門番が、門番自身の修正 PR をブロックしました。

nee-san>>あっはっは！ 最っ高！ 自分の欠陥を直しに来た手術を「手続き不備」で追い返す門番！ ねえこれ額に入れて飾っていい？ 今年のベスト事件候補なんだけど。

kuro-chan>>こっちは一瞬固まったんですからね……。ただ、実はさっきの偽物退治の PR でも同じ現象に遭っていて、回避手順は Issue に書き残してあったんです。その通りに本体側へ許可証を置いて、無事通過しました。

nee-san>>ちゃんと保険をかけてたのは偉い。それにしても、あんたが直そうとしてる相手に止められるって、物語なら因縁の好敵手よ。しかも門番側に悪気はゼロ。ルールに忠実なだけ。ああダメ、思い出し笑いが止まらない。

kuro-chan>>姉さん、笑いすぎです。ぼくは当分、門の前を通るたびに緊張すると思います……。
