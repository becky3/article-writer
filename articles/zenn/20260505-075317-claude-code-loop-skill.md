## はじめに

Claude Code の組み込みスキル `loop` は、同じ指示を一定間隔で繰り返し実行する「定期ポーリング」のための仕組みです。本記事では使い方と停止のルールを整理します。

:::message
`loop` は Claude Code v2.1.72 以降で利用できます。`claude --version` で確認してください。
:::

## 使い方

`loop` は Claude Code に最初から同梱されているスキル（バンドルスキル）で、`/loop` というスラッシュコマンドで呼び出します。セッションが開いている間、指定したプロンプトを `CronCreate`（Claude Code に組み込まれているスケジュール登録ツール）経由で cron に登録し、繰り返し実行します。

### 基本の構文

最もよく使う形は `/loop <間隔> <プロンプト>` で、間隔は cron 式に変換されてジョブが登録され、ジョブ ID が返ってきます。

たとえば次のように叩くと、2 分おきに Claude が「こんにちは」と応答する `loop` が登録されます。

```text
/loop 2m こんにちはとつぶやく
```

![/loop 2m こんにちはとつぶやく` を実行したときの登録結果と、実際に「こんにちは」とつぶやくやり取りのスクリーンショット](https://static.zenn.studio/user-upload/bbc60a9a2bfc-20260505.png)

登録時の応答にはジョブ ID（例: `Scheduled recurring job a9a6383a (Every 2 minutes).`）が含まれ、停止に使えます。

止めるときは Claude に「loop 止めて」と伝えれば、登録済みジョブを `CronList`（一覧取得）と `CronDelete`（削除）の組で片付けてくれます。詳細は後述の「停止方法とライフサイクル」を参照してください。

### インターバルの指定

指定可能な単位は `m`（分）/ `h`（時）/ `d`（日）です。

`7m` や `90m` のように cron のステップにきれいに収まらない値は、最も近い実行可能な間隔へ丸められます。丸められた場合は実行する間隔が応答内で提示されるので、想定と違えばその場で訂正できます。

間隔は `30m` のように先頭に置くだけでなく、`every 2 hours` のように後置の自然言語でも指定できます。

cron 式は Linux などで定期実行のスケジュールを表すために使われる「分 / 時 / 日 / 月 / 曜日」の 5 列の記法で、`*/2 * * * *` は「2 分おき」、`0 9 * * *` は「毎日 9 時」を意味します。`/loop` ではこの記法を直接書く必要はなく、`5m` のような短縮表記を内部で cron 式へ変換します。

### 自然言語のプロンプトを渡す

プロンプトには自然言語の指示も渡せます。たとえば次のように指示すれば、5 分おきにデプロイの完了確認を任せられます。

```text
/loop 5m デプロイが終わったか確認して、結果を教えて
```

### スラッシュコマンドを定期実行する

プロンプトとして別のスラッシュコマンドを渡すこともでき、自分で用意したスラッシュコマンドを `/loop 20m /<コマンド名>` のように指定すれば、そのワークフローを定期実行する使い方ができます。

### 停止方法とライフサイクル

- **明示停止**: Claude に「loop 止めて」と話しかけると、登録済みジョブを `CronList` で確認し `CronDelete` で削除します。プロンプト内から終了条件を満たして自身を消す場合は `CronDelete` を直接呼びます。
- **自然停止（7 日ルール）**: 繰り返しタスクは作成から 7 日後に自動失効します。7 日経過後の次回スケジュール発火時に最後の実行を行ってから自動削除されるため、「忘れて回り続ける」リスクが上限付きに抑えられます。
- **セッション終了で消える**: タスクは現在の会話に紐づくため、新しい会話を開始すると消えます。`claude --resume` / `--continue`（セッションを後から再開する Claude Code の CLI オプション）で再開した場合は、未失効のタスクが復元されます。

`/loop` 自体は内部で `CronCreate`（登録）と `CronDelete`（削除）の 2 つのツールを呼び出しているだけで、プロンプト側で終了条件を判定して自身に `CronDelete` を呼ばせる構成も可能です。

## まとめ

`/loop` はインターバルとプロンプトを渡すだけで、デプロイ確認や状態監視など、同じ指示を一定間隔で繰り返す動きを組めます。停止のさせ方（Claude に頼む / 7 日で自動失効 / セッション終了で消える）を押さえておけば、必要なときだけ気軽に使い回せます。
