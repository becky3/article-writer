# 記事執筆ワークフロー

`/write-zenn` スキルを使った記事執筆と、その過程で得られたフィードバックをスキル/ルール/テンプレート/エージェントに還元する標準的な作業フロー。

## 基本原則

1 Issue で **「記事ファイルの追加」と「執筆中に受けたフィードバックを `.claude/skills/write-zenn/` 配下へ反映するルール/スキル修正」をセットで扱う** ことを標準とする。

### 背景

記事生成は `/write-zenn` スキルが行うが、出力された記事に対する人間レビューでは似た指摘が繰り返される傾向がある。指摘を記事 1 本だけに反映しても次の記事生成時に再発する。**指摘 → ルールへの還元** を作業の一部として組み込むことで、スキルの品質をインクリメンタルに高める。

## 標準フロー

1. Issue を起票（記事案 / 改善案）
2. ブランチを切って `/write-zenn` で記事を生成
3. 人間レビュー → 修正サイクルを繰り返す
4. **PR 作成前** に Claude が修正サイクルから「繰り返し出た指摘」「生成時点で気付けなかった構成・体裁の問題」を抽出し、ユーザーに提示する
   - 抽出の素材: 人間レビューでの指摘・修正コミット差分・チャット履歴
5. ユーザーが合意した観点を以下のいずれかに反映する
   - `.claude/skills/write-zenn/SKILL.md`（処理手順・確認ポイント）
   - `.claude/skills/write-zenn/quality-guidelines.md`（執筆品質ガイドライン）
   - `.claude/skills/write-zenn/template-*.md`（記事タイプ別テンプレート）
   - `.claude/agents/repo-scanner.md`（候補抽出ロジック）
6. 記事 + ルール修正をまとめてコミットし、1 PR でレビュー

## 例外

純粋な誤字修正等で「ルールに反映する観点が無い」場合に限り、記事のみの PR を許容する。判断に迷う場合はルール修正を含める方向で運用する。
